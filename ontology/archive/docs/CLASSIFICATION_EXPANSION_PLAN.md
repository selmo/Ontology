# 분류 확장 계획서 (시나리오 B: 적극적 확장)

**작성일:** 2025-12-10
**버전:** 1.0
**목표:** 12개 도메인, 350개 분류

---

## 1. 확장 개요

### 1.1 현황 → 목표

| 항목 | 현재 (v2.1) | 목표 (v3.0) | 증가율 |
|------|-------------|-------------|--------|
| 도메인 | 10개 | 12개 | +20% |
| 대분류 | 10개 | 12개 | +20% |
| 중분류 | 40개 | 60개 | +50% |
| 소분류 | 125개 | 278개 | +122% |
| **총 분류** | **175개** | **350개** | **+100%** |

### 1.2 확장 원칙

1. **표준 기반 확장**: 국제/국내 표준 분류체계 우선 적용
2. **균형 잡힌 분포**: 도메인당 최소 25개, 최대 40개 분류
3. **계층 일관성**: 모든 도메인에서 대-중-소 3단계 구조 유지
4. **하위 호환성**: 기존 ID 유지, 신규 ID만 추가
5. **용어 보류**: 이번 확장에서 용어(Term)는 변경하지 않음

## 4. ID 체계 및 마이그레이션

### 4.1 ID 체계 (유지)

```
현행 9자리 체계 유지:
C + DD + LL + SSSS
│   │    │    └── 일련번호 (4자리)
│   │    └── 중분류 (2자리)
│   └── 도메인 (2자리)
└── 타입 (C: Classification)

예시:
C01010001 = 01(공공행정) + 01(정부조직) + 0001(첫번째)
C08010001 = 08(디지털커머스) + 01(이커머스플랫폼) + 0001(첫번째)
C11020003 = 11(과학기술) + 02(자연과학) + 0003(물리학)
```
