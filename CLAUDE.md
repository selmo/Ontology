# CLAUDE.md

이 파일은 Claude Code (claude.ai/code)가 본 저장소에서 작업할 때 참고하는 가이드입니다.

## 저장소 개요

다양한 도메인의 참고 문서를 관리하는 **문서 저장소**입니다:

- **commerce/** - 이커머스 및 디지털 경제 관련 보고서
- **finance/** - 금융 정책 및 규제 문서
- **law/** - 법률 판례 및 세제 관련 문서
- **medical/** - 의료 및 건강 가이드라인 문서
- **public/** - 정부 부처 업무계획 및 정책 문서
- **ontology/** - 분류 및 용어 관리 체계

문서 형식: PDF, HWPX

## 온톨로지 체계

`ontology/` 디렉토리는 문서 분류를 위한 체계적인 분류·용어 시스템을 포함합니다:

### 주요 파일
- `context.txt` - **기준 파일**. MC 분류·용어 통합 체계 개요
- `mc_clsf.sql` - 분류 카테고리 SQL INSERT문 (MC_CLSF 테이블)
- `mc_term.sql` - 용어 정의 SQL INSERT문 (MC_TERM 테이블)
- `context.xlsx` - **deprecated** (사용하지 않음)

### ID 체계

**분류 ID (CLSF_ID)**
```
C + DD(도메인) + LL(중분류) + SSSS(일련번호)
```
- 총 9자리: `C` + 2자리 도메인 + 2자리 중분류 + 4자리 일련번호
- 예: `C01010001` = 도메인 01 + 중분류 01 + 일련번호 0001

**용어 ID (TERM_ID)**
```
T + DD(도메인) + LL(중분류) + SSSS(일련번호)
```
- 총 9자리: `T` + 2자리 도메인 + 2자리 중분류 + 4자리 일련번호
- 예: `T01010001` = 도메인 01 + 중분류 01 + 일련번호 0001

### 데이터베이스 스키마

**MC_CLSF (분류)**
```sql
CLSF_ID     VARCHAR2(32)   -- 분류 ID (예: C01010001)
CLSF_NAME   VARCHAR2(500)  -- 분류명
CLSF_DESC   VARCHAR2(4000) -- 분류 설명
PRT_CLSF_ID VARCHAR2(32)   -- 상위 분류 ID
GROUP_YN    CHAR(1)        -- 그룹 여부 (Y: 상위, N: 하위), 기본값 'N'
DELETE_YN   CHAR(1)        -- 삭제 여부, 기본값 'N'
README      CLOB           -- 비고 (상세 설명)
```
- 최상위 `PRT_CLSF_ID`: `03facd74b2d24f7cab807b8980391649`

**MC_TERM (용어)**
```sql
TERM_ID      VARCHAR2(32)   -- 용어 ID (예: T01010001)
TERM_NAME    VARCHAR2(500)  -- 용어명 (한글)
TERM_NAME_EN VARCHAR2(500)  -- 영문명
ACRONYM      VARCHAR2(100)  -- 약어 (예: OMO, D2C)
TERM_DESC    VARCHAR2(4000) -- 용어 설명
PRT_TERM_ID  VARCHAR2(32)   -- 상위 용어 ID
TERM_TYPE    CHAR(1)        -- 용어 유형 (T: 일반 용어), 기본값 'T'
DELETE_YN    CHAR(1)        -- 삭제 여부, 기본값 'N'
README       CLOB           -- 연결 분류 ID (예: C01010001)
```
- 최상위 `PRT_TERM_ID`: `4147179070a84d3887b97eb57085d850`

### SQL INSERT 간소화

기본값이 있는 필드는 INSERT에서 제외하여 간소화:

**MC_CLSF 예시**
```sql
INSERT INTO MC_CLSF (CLSF_ID, CLSF_NAME, CLSF_DESC, PRT_CLSF_ID, GROUP_YN, README)
VALUES ('C01000001', '공공', '공공 부문 정책 및 행정 전반', 'ROOT_ID', 'Y', '도메인 01 설명');
```

**MC_TERM 예시**
```sql
INSERT INTO MC_TERM (TERM_ID, TERM_NAME, TERM_NAME_EN, ACRONYM, TERM_DESC, PRT_TERM_ID, TERM_TYPE, README)
VALUES ('T02010003', '공개시장운영', 'Open Market Operations', 'OMO', '중앙은행이 유가증권을 매매하여 통화량을 조절하는 정책수단', 'ROOT_ID', 'T', 'C02010001');
```

## 작업 가이드

### 새 분류 추가
1. context.txt의 ID 체계 준수: `C + DD + LL + SSSS`
2. 도메인 최상위: `CDD000001` (예: C01000001)
3. 중분류: `CDDLL0001` (예: C01010001)
4. 소분류: 같은 중분류 내 일련번호 증가 (예: C01010002)
5. `GROUP_YN`: 하위 항목이 있으면 `'Y'`, 없으면 `'N'`
6. `README`: 분류의 목적과 범위를 상세히 기술

### 새 용어 추가
1. context.txt의 ID 체계 준수: `T + DD + LL + SSSS`
2. 해당 분류의 중분류 코드(LL) 사용
3. `TERM_NAME`: 한글 용어명
4. `TERM_NAME_EN`: 영문 용어명
5. `ACRONYM`: 약어가 있는 경우 기록 (예: OMO, D2C, GERD)
6. `TERM_DESC`: 한글 설명
7. `README`: 연결 분류 ID 기록 (예: `'C01010001'`)

### 파일 동기화
- **context.txt**가 기준 파일
- mc_clsf.sql, mc_term.sql은 context.txt와 동기화 유지
- context.xlsx는 사용하지 않음 (deprecated)
