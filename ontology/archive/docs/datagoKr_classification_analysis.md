# data.go.kr 실제 데이터셋 기반 분류 체계 분석

## 분석 방법
실제 공개된 데이터셋 제목과 내용을 분석하여 중분류/소분류 패턴 추출

## 결론

### 중분류 구조 제안

각 도메인별 3-4개 중분류를 두고, 중분류 아래 2-4개 소분류 배치

**구조:**
```
도메인 (CDD000001)
├─ 중분류1 (CDDLL0001)
│  ├─ 소분류1 (CDDLL0002)
│  └─ 소분류2 (CDDLL0003)
├─ 중분류2 (CDD(LL+1)0001)
│  └─ 소분류1 (CDD(LL+1)0002)
└─ 중분류3 (CDD(LL+2)0001)
```

### 적용 방안

1. 실제 데이터셋이 풍부한 도메인: 데이터 기반 분류
   - 공공행정, 교육, 보건의료, 재정금융

2. 데이터셋이 부족한 도메인: 표준 분류체계 적용
   - 사회복지, 법률, 재난안전, 문화관광, 환경기상

3. 기존 데이터 유지 도메인: 프로젝트 문서 기반
   - 산업경제 (이커머스 특화)
